Before proposing anything, interview me. Ask at least 5 questions:
- What problem does this package solve?
- Who uses it? (just me, my team, public)
- What are the inputs and outputs?
- What existing tools or libraries should it integrate with?
- What are the hard constraints (performance, compatibility, format)?
- Is this research code or production code?

1. ideate — explore 2-3 approaches, present Decision Summary. Gate.
2. architect — domain model, module table, boundary decisions. Gate.
3. design — file structure, protocols, config. Gate.
4. scaffold — generate all files with TODO bodies and smoke tests.

Do not skip any step. Do not combine steps. Gate at each one.
